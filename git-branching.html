Git Branching & Merging

Think of source code as a train track. Instead of everyone working on the main track (and causing crashes), developers build switch tracks (branches) to build their features safely. Watch how code branches off and merges back together.

👆 Select a scenario to watch the Git train in action.
🛤️ Main Track (main)
The main branch is the primary track. This is usually the code running in production. No one should commit broken code here! It must always stay stable.
🔀 Switch Track (branch)
A feature branch is a parallel track. You can build, break, and test your new code entirely isolated from the main track. Once perfect, you merge it back.
🚃 Train Cars (commits)
Each commit is a train car carrying a specific set of file changes. It also has an ID (like a1b2) and it links to the car immediately behind it, forming the unshakeable chain of history.
